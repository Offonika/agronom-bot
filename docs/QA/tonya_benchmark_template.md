# Шаблон benchmark для Тони (20–50 кейсов)

Цель: собрать эталонные кейсы и быстро увидеть, где бот ошибается чаще всего.

## Как заполнять
- 1 строка = 1 кейс.
- Кейс должен содержать 3–8 фото (минимум: общий вид + лист лицо + лист изнанка).
- Для бага прикладывать 2–3 последовательных скрина: `нажато -> отправлено -> ответ бота`.
- Время фиксировать в МСК.
- Для Google Sheets: `Файл -> Импорт -> Загрузить -> Разделитель: запятая`.

## Быстрый режим через QA-группу Telegram
- В QA-группе (ID задается в `QA_INTAKE_CHAT_ID`) можно отправлять кейсы в бот командой `/qa`.
- Основной путь: `/qa` и заполнить карточку кнопками (сценарий, тип ошибки, критичность).
- Доп.поля (опционально): растение, ожидаемое, фактическое, комментарий, уверенность, diagnosis_id.
- Скрины/фото можно дослать в активной сессии `/qa`, бот прикрепит их к кейсу.
- Быстрый fallback: сразу одной строкой:  
  `Алоэ | followup_photo | S2 | context_lost | Ожидали уточнение полива | Бот потерял контекст | после досылки фото`
- Можно отправить скрин/документ с подписью, начинающейся с `#qa ...`.
- Бот сам проставляет тестера и время (МСК), сохраняет свободный комментарий (`notes`/`raw_text`) и пытается мягко привязать кейс к диагнозу (`diagnosis_id`).
- Кейс сохраняется как структурированное событие в БД (`beta_events.event_type = qa_case_logged`).
- Для выгрузки в таблицу используйте `docs/QA/qa_intake_export.sql`.

## Поля
| Поле | Что писать |
|---|---|
| `ID кейса` | ID кейса/диагноза (если есть) |
| `Telegram ID тестера` | Telegram ID тестера |
| `Дата/время (МСК)` | Дата и время теста в МСК |
| `Растение` | Что за растение (минимум бытовое название) |
| `Сценарий` | `new_diagnosis` / `followup_photo` / `ask_products` / `assistant_chat` / `pending_retry` |
| `Кол-во фото` | Сколько фото отправлено |
| `Эталон (ground truth)` | Эталон от агронома (что реально произошло) |
| `Краткий ответ бота` | Коротко, что ответил бот |
| `Уверенность` | Уверенность из ответа бота, если есть |
| `Тип ошибки` | `none` / `wrong_class` / `wrong_risk` / `wrong_action` / `context_lost` / `ux_dead_end` / `hallucination` |
| `Критичность` | `S1` критично / `S2` заметно / `S3` косметика |
| `Ожидаемое поведение` | Что должно было быть |
| `Фактическое поведение` | Что получилось фактически |
| `Шаги воспроизведения` | Короткие шаги воспроизведения (1–5) |
| `Скриншоты` | Ссылки на скрины/папку |
| `Заметки` | Любые наблюдения |

## Минимальные квоты по типам (для 20+ кейсов)
- `new_diagnosis`: 8+
- `followup_photo`: 4+
- `ask_products`: 3+
- `assistant_chat`: 3+
- `pending_retry`: 2+

## Приоритизация фиксов
- `S1`: может привести к вредной рекомендации или блокирует основной сценарий.
- `S2`: сценарий выполняется, но с ошибкой/путаницей.
- `S3`: мелкие UX/текстовые проблемы без риска.

## Быстрый чек-лист для ручного прогона
1. Ошибка анализа не сбрасывает фото-сессию.
2. `Спросить про препараты` + регион текстом дает полезный ответ сразу.
3. Короткие ответы про воду/грунт (`мокро/сухо`, состав грунта) понимаются в контексте.
4. Low-confidence (<65%) запускает re-check и просит минимум 2 фото.
5. Досылка фото к активному растению сохраняет нить кейса.
6. Pending/очередь не зависает навсегда, есть продолжение после retry.
