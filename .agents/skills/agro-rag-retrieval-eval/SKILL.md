# Agro RAG Retrieval Eval

## Scope

- Применяй skill для изменений, влияющих на поиск: chunking, индексация, embeddings, rerank, запросный prompt retrieval.
- Не применяй для чисто UI/бот-правок без влияния на retrieval.

## Workflow

1. Определи baseline и тестовый срез:
- фиксированный набор эталонных вопросов по houseplants
- контрольный индекс/конфигурация для сравнения
2. Запусти оценку метрик retrieval:
- hit-rate@k
- recall@k
- MRR/nDCG (если доступны)
3. Сравни с baseline:
- выдели улучшения/регрессии
- проверь worst-case запросы (низкая уверенность, редкие болезни)
4. Зафиксируй пороги качества для merge:
- блокировать merge при регрессии выше допустимого порога
- допускать merge при нейтральном/положительном результате
5. Сформируй итоговый отчёт:
- метрики до/после
- список деградировавших запросов
- рекомендации по chunking/фильтрам/rerank

## Guardrails

- Не делай выводов по 2-3 случайным запросам: используй стабильный eval-набор.
- Не сравнивай результаты на разных датасетах без явной пометки.
- При отсутствии eval-набора зафиксируй это как риск качества.
