# Agro Doc Sync

## Scope

- Применяй skill, когда меняется поведение продукта, API, flow бота, ограничения, валидации или схема данных.
- Не применяй для чисто технических правок без изменения поведения (например, форматирование, рефакторинг без изменений логики).

## Workflow

1. Определи, что именно изменилось:
- API-контракт (`app/controllers`, `app/models`, `openapi/openapi.yaml`)
- Бизнес-правила (`app/services`, `bot/flow`, `worker`)
- Данные/ограничения (`migrations`, SQL constraints)
2. Сопоставь изменения с обязательными документами:
- `docs/srs.md` — scope/FR и пользовательское поведение
- `docs/data_contract.md` — поля, валидаторы, лимиты, обязательность
- `docs/payment_flow.md` — если затронуты оплаты/autopay/webhook
- `docs/prd.md` — продуктовые сценарии/ожидания
- `CHANGELOG.md` — semver-запись
3. Внеси минимальные правки, без переписывания несвязанных разделов.
4. Зафиксируй соответствие текста и реализации:
- терминология в docs совпадает с ключами/enum/кодами в коде
- лимиты (например фото 3-8) и обязательные шаги совпадают с runtime-проверками
5. Добавь в итоговый отчёт список обновлённых документов и что именно синхронизировано.

## Guardrails

- Не оставляй изменения поведения без отражения в docs.
- Не добавляй новые требования в docs, если они не реализованы в коде.
- Не удаляй исторически важные разделы без запроса пользователя.
